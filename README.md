# 📈 주식 통합 알리미 봇 — 사용 설명서

서은님 텔레그램으로 **한국시간 8~22시 매시 정각**, 모니터링 중인 **20개 텔레그램 방의 지난 1시간 글**을 모아 분석해 보내주는 봇입니다. GitHub Actions가 자동으로 돌려서 내 컴퓨터를 켜둘 필요가 없습니다.

---

## 1. 이 봇이 보내는 것

매시간 텔레그램 메시지 한 통이 오고, 두 부분으로 돼 있습니다.

### 📊 감시종목 thesis 점검 (상단)
서은님 투자 thesis 기준 점검입니다. **침묵이 기본값** — 보유·관심 종목에 실제 변화(대형 주문·수주·가이던스·마진·희석·실적 등)가 있을 때만 정해진 형식으로 보고하고, 없으면 `✅ 이상 없음` 한 줄만 표시합니다.

### 📝 텔레그램 분석 (하단, 최근 1시간)
20개 방의 지난 1시간 글을 읽고 정리합니다.

- **🌡️ 센티먼트** — 공포(-100) ~ 탐욕(+100)
- **🎯 이 시간의 핵심** — 가장 중요한 변화 2~3줄
- **📊 시장·매크로** — 지수·유가·금리·지정학 흐름(글에 나온 것만)
- **💎 보유·관심 섹터 영향** — 메모리·전력·광통신·넥스트테크·GOOGL 등 **내 섹터별**로 정리
- **📰 신규 부상** — 서로 다른 2개+ 채널이 동시에 언급한 새 테마
- **⚠️ 점검 포인트** — 이번 시간에 확인·모니터링할 것 (매수/매도 지시는 안 함)
- **🗣️ Top 키워드** / **🔥 인기 게시글 TOP 10**(각 글 1~2줄 요약)

> 모든 내용의 **출처는 그 20개 텔레그램 방의 글**입니다. 외부 웹 검색은 기본 꺼져 있고, AI는 글에 있는 사실만 요약합니다(없는 수치 지어내지 않음).

## 2. 어떻게 돌아가나 (흐름)

```
한국시간 8~22시, 매시 정각
   │
GitHub Actions 가 자동 실행
   │
1) 20개 방의 최근 70분 글 수집 (내 텔레그램 계정 세션)
2) 감시종목 thesis 점검 (변화 있을 때만 보고)
3) 섹터별 시장 분석 (Gemini AI)
4) 내 텔레그램 봇으로 전송  →  폰·모든 기기에 알림
```

- **무료**입니다. (GitHub Actions + Gemini 무료 등급)
- 알림은 한 곳(GitHub)에서만 돌지만, 텔레그램이 **모든 기기로 동기화**해줍니다.

## 3. ⭐ 바꾸고 싶을 때 (관리 가이드)

모든 변경은 **GitHub에서 해당 파일 열기 → 연필(✏️ Edit) → 수정 → Commit changes** 면 끝입니다. 다음 정각 실행부터 반영돼요. (바로 보고 싶으면 Actions 탭 → Run workflow)

### ① 감시 채널 바꾸기 → `channels.txt`
한 줄에 채널 하나. 채널 ID(`-100...`) 또는 `@username`.
- **추가**: 새 줄에 ID나 `@아이디` 입력
- **잠깐 끄기**: 줄 맨 앞에 `#`
- `#` 뒤는 메모(채널 이름)라 봇이 무시해요
- ⚠️ 내 텔레그램 계정이 **그 방에 가입돼 있어야** 읽힙니다

### ② 리포트의 "💎 보유·관심 섹터" 바꾸기 → `watchlist.py` 의 `REPORT_SECTORS`
파일 위쪽 `REPORT_SECTORS` 목록을 고치면 돼요. 형식: `("섹터이름", "관련 종목·키워드")`.
새 섹터 추가 예시:
```python
("로봇/휴머노이드", "테슬라 옵티머스, 레인보우로보틱스, 두산로보틱스"),
```
이 섹터 이름·키워드를 보고 AI가 그 시간 글을 섹터별로 묶어 요약합니다.

### ③ "📊 감시종목 thesis 점검" 대상 바꾸기 → `watchlist.py` 의 `HOLDINGS`
보유/관심 종목 목록이에요. 각 항목에서:
- `constituents` : 글에서 **이 단어들이 잡히면** 그 종목 점검 대상 (예: `"SK하이닉스", "HBM"`)
- `kill` : kill condition 메모
- `check` : `False` 면 일상 점검 제외(코어·현금성)
- `focus` : 무엇을 볼지 메모

종목 추가 예시(한 칸 복사해서 값만 바꾸기):
```python
{
    "ticker": "000000", "name": "새종목", "region": "KR", "check": True,
    "constituents": ["새종목", "관련키워드"],
    "focus": "무엇을 감시할지",
    "kill": "어떤 조건이면 위험한지",
},
```

### ④ thesis 점검 기준·말투 바꾸기 → `thesis_prompt.txt`
"이상 없음 vs 보고" 판단 규칙, 보고 형식, 알림/비알림 기준이 여기 들어있어요. 그대로 고치면 AI 판정에 바로 반영됩니다.

### ⑤ 실행 시간 바꾸기 → `.github/workflows` 의 워크플로 파일
`cron` 줄을 바꿉니다. GitHub은 **UTC** 라 한국시간 −9.
- 현재(8~22시 매시간): `cron: '0 23,0-13 * * *'`
- 24시간 매시간: `cron: '0 * * * *'`
- 2시간마다(8~22시): `cron: '0 23,1,3,5,7,9,11,13 * * *'`

### ⑥ 리포트 종류·개수 바꾸기 → Secrets `ENV_FILE`
GitHub **Settings → Secrets and variables → Actions → ENV_FILE** 수정.
- `REPORT_MODE` : `full`(thesis+시장) / `thesis_only`(thesis만) / `alert_only`(변화 있을 때만)
- `TOP_POSTS` : 인기 게시글 개수
- `TOP_KEYWORDS` : 키워드 개수
- `LOOKBACK_MINUTES` : 몇 분치 글을 볼지(매시간이면 70 권장)

## 4. 비밀값(Secrets) — 보안

키·토큰·세션은 코드가 아니라 GitHub **Secrets(`ENV_FILE`)** 에 암호화 저장돼 있어요. 공개 저장소여도 남이 못 봅니다.
- **절대** `.env` 파일을 저장소에 올리지 마세요(키 노출).
- 봇 토큰을 새로 발급하고 싶으면: 텔레그램 `@BotFather` → `/revoke` → 새 토큰을 `ENV_FILE` 의 `BOT_TOKEN` 에 교체.
- Gemini 키 교체: aistudio.google.com/apikey 에서 재발급 → `ENV_FILE` 의 `GEMINI_API_KEY` 교체.

> 🔒 내 종목·전략 노출이 싫으면 저장소를 **Private** 로 두세요(Settings → Danger Zone → Change visibility). 단 Private는 무료 실행시간이 월 2000분이라 매시간이면 빠듯할 수 있어요(그땐 2시간마다로).

## 5. 안 올 때 / 문제 해결

1. **리포트가 안 와요** → GitHub **Actions** 탭에서 최근 실행이 초록 ✓ 인지 확인. 빨간 ✗ 면 클릭 → 빨간 글씨 확인.
2. **특정 방이 누락돼요** → 내 텔레그램 계정이 그 방에 **가입**돼 있는지 확인.
3. **"이상 없음"만 계속 와요** → 정상이에요(thesis는 침묵이 기본). 시장 분석은 그 아래에 항상 옵니다.
4. **실행이 멈췄어요** → 저장소를 **60일간 안 건드리면** GitHub가 스케줄을 일시정지해요. 아무 파일이나 한 번 수정·커밋하면 다시 켜져요.
5. **시간이 정각보다 늦어요** → 무료 스케줄 특성상 5~15분 지연될 수 있어요(정상). 칼정각이 필요하면 VPS로 이전.

## 6. 파일 구조 (뭐가 뭔지)

```
stock-alert-bot/
├─ channels.txt          ← ① 감시 채널 20개
├─ watchlist.py          ← ②③ 리포트 섹터 + thesis 감시종목
├─ thesis_prompt.txt     ← ④ thesis 판정 기준
├─ .github/workflows/    ← ⑤ 실행 시간(cron)
│   └─ hourly.yml
│
├─ collector.py          텔레그램 글 수집
├─ analyzer.py           섹터별 AI 분석
├─ thesis_monitor.py     감시종목 thesis 점검
├─ formatter.py          리포트 메시지 구성
├─ sender.py             봇 전송
├─ llm.py                Gemini 호출
├─ config.py             설정 로더
├─ main.py               1회 실행(=한 시간치)
├─ requirements.txt      라이브러리 목록
│
├─ setup_colab.ipynb     (최초 1회) 세션·CHAT_ID 만든 브라우저 노트북
├─ gen_session.py        세션 생성 스크립트
└─ scheduler.py / run.sh / setup.sh / deploy/   ← (선택) VPS·내컴퓨터로 돌릴 때
```

## 7. 주의 / 한계

- 이 봇은 **보조 도구**입니다. thesis 보고와 시장 요약은 텔레그램 글 기반 **추론**이고, 직접적인 매수/매도 지시는 하지 않아요. 최종 판단·공시 원문 확인은 직접 하세요.
- Gemini 무료 등급이라 글이 아주 많은 시간대엔 분석에 1~2분 걸릴 수 있어요(정상).
- 종목명·요약은 AI가 만들어 드물게 어색할 수 있어요 — 이상하면 `analyzer.py` 의 프롬프트나 `thesis_prompt.txt` 를 조이면 됩니다.
